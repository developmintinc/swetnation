Getting startedTop

Quick start guide

The quick start guide only covers the most important steps to get your template up as quickly as possible.

1. Change your logo in index.html

The existing logo is located at images/logo.png, you can replace that file with your own logo file with the same name.

If your logo has a different file extension you will need to change the HTML code of the page, so do a search in the code for the term <!--Logo--> and you should find the <img> tag for the logo. Change the src="" attribute to point to your new logo.

You may need to adjust the padding around your logo to centre it within the logo area. To do this go into styles.css and in section 1 locate .logo-img, adjust the padding to suite your logo.

2. Configure the forms to send form data to your email address

Note: these steps should be completed for both the config.php and config-register.php files

The contact form is an iPhorm, which is one of our CodeCanyon items that we sell separately. The documentation for the iPhorm is in the documentation/form-documentation folder, there is also a link to it at the bottom of this section.

To set up the form the only step you need to do is Step 4 of the form installation guide which is to change the form to send enquiries to your email address. I've copied these instructions below.

To configure the contact form, open contact-form/config.php and on line 19 you will find the current email address that will be receiving the enquiries. Change that to your email address as shown below.

$recipients = array(
    'spam@themecatcher.net'
);

becomes

$recipients = array(
    'your@emailaddress.com'
);

While you are in contact-form/config.php you can also change the success message if you wish, which is displayed when the form is successfully submitted. The code for that is on line 6.

To customise the form further or for form troubleshooting, see the form help.

3. Enter your launch date in js/scripts.js

You can change the launch date shown inside the slider by locating the comment // Enter your launch date in js/scripts.js. The default code is shown below, you can just change the relevant numbers of the date to match your launch date.

launchDate = {
    day: 18,
    month: 3,
    year: 2012,
    hour: 12,
    min: 0,
    sec: 0  
};

That is all for the Quick start guide, for further customisation help see the more detailed instructions in the sections below.

HTML files

The HTML file for the template is in the template folder. Each part of the file is commented to help you find the section you want to make changes to. The content area for each page begins with <div class="content-inner">.

CSS files

Launch Pad has one main style sheet css/styles.css and this is the file that you should be making any changes to. The file is split into sections by numbered comments and the file has a table of contents to help you find the right section.

JavaScript files

There are 3 JavaScript files used in most of the HTML files, located in the js folder:

  • 1. jQuery - js/jquery-1.5.2.min.js - The jQuery JavaScript library
  • 2. Plugins - js/plugins.js - All third-party plugins & scripts
  • 3. Custom scripts - js/scripts.js - Custom JavaScript

Adding your own JavaScript

If you want to add your own JavaScript, you can do so in js/scripts.js, if you need to add code that executes after the DOM is ready you can add it after this line:

$(document).ready(function() {

To add your own third party plugin just add a link to the JavaScript file in the head of the document. If you placed your plugin in the js folder, you would add this code inside the <head> tags of the document:

<script type="text/javascript" src="js/your_plugin.js"></script>

PSD files

You will find the PSD files in resources/photoshop, each element is grouped neatly in their own folders within the PSD. There are 2 main PSDs, one for light design and the other for dark. You will also find 2 more PSDs required if you decided to make your own colour scheme, the file names represent the image file name.

Changing the navigation / Tabs

To find the navigation in index.html search for <!--Tab links-->. On the lines below that you'll see the navigation links that you can change or remove to suit.

Add new Tab

To add a new tab you will need to firstly create a new tab button. Each tab button is numbered to correlate with the div which it triggers. If you are adding the another tab you would add into the list a new li like this: 

<ul>
<li><a href="#content">Info</a></li> 
<li><a href="#tasks">Tasks</a></li> 
<li><a href="#video">Video</a></li> 
<li><a href="#gallery">Gallery</a></li>
<li><a href="#contact">Contact</a></li> 
<li><a href="#custom">Custom</a></li>  
</ul>

You will now need to add a new content area for the #custom tab with an ID of custom. Create a new div with these parameters and add your content inside.

<div id="custom">your content here</div>

Remove a Tab

To remove a tab simply remove the relevent li line from the list/ menu of tabs like below. Then find the div with the matching ID name and delete the div and its contents.

<ul>
<li><a href="#content">Info</a></li>

<li><a href="#video">Video</a></li>

Translating or changing the month names

Configuring the progress bar

You can change the progress percentage amount to change how far the bar animates inside the bar container. Open js/scripts.js and search for the term progressPercentage and change the number to the percentage you would like.

Changing the animation speed

You can also change the speed of the progress bar animation. Find the term progressAnimationSpeed in js/scripts.js change progressAnimationSpeed = 2000 to the speed you would like. The speed is measured in milliseconds, so 2000 means the animation takes 2 seconds from start to finish. Increasing the number will mean the animation takes longer and vice versa.

Changing the footer menu

There is a menu inside the left footer column which are links to activate information pop ups.

Adding a footer popup box

You will need to first add a new menu item to the footer menu for your new box, you will need a unique lowercase name for it. For example, I will call my new popup box "guestbook". To add the menu item, search for the ul with the class footer-menu and add a new list item to it, just copy and paste an existing <li>:

<li><a class="footer-pop-out-trigger" title="Show / Hide" id="pop-out-trigger-guestbook" href="#">
    <span class="up-ico">Guestbook</span></a></li>

Note: the first highlighted text I have change the word to my unique name for the popup box. The second highlighted text is what you want to appear in the footer menu that visitors will see.

Next, you need to add the popup box itself. It's easier to copy the HTML for an existing popup box, if you search for the term footer-pop-out-box you will find one. Once you have copied the code, you should change the two highlighted areas below to your unique popup box name:

<div id="guestbook-pop-out" class="footer-pop-out-box guestbook-pop-out-box">

Removing a footer popup box

Search for the ul with the class footer-menu and remove the li and the link within them which you do not require.

It is not essential to remove the pop up boxes if you have removed the link. However, if you would like to then locate the comment tag containing the pop up box and delete all the content between the comments. For example to remove the Twitter locate the <!--Hidden pop out boxes - Twitter--> and remove everything under it until the next pop up box comment.

Adding your social networking links

To find the HTML code for the social networking "Connect with me" popup box, search index.html for the term <!--Hidden pop out boxes - Social icons-->, this will land you at the list of icons.

To find the HTML code for the social networking mini icons on the content footer (which may be hidden - see design options above), search index.html for the term <!-- CONTENT FOOTER , this will land you at the list of icons.

There is one social networking link and icon per line. You should be able to tell what this link is for by looking at the name of the image file or the alt tags.

Removing a link

To remove a link, delete the entire line with the link and image on it.

Adding a link

To add a link, copy the code for an existing icon and paste it onto a new line, then change the image src attribute to point to the icon of the social network or service. We have included a lot of icons in the images/social-icons/ folder. You should also change the href attribute of the <a> tag to point to the URL of your account. Here is a complete line for example (only on multiple lines for readability):

<div class="one-icon-wrap"><a href="http://www.facebook.com/your-username">
<img src="images/social-icons/facebook.png" width="32" height="32"
alt="Facebook" /></a></div>

Note: you will need to make the change to the HTML source code on all the other pages you will be using on your site.

Adding your Twitter feed

In the file js/scripts.js, near the top of the file you should find the code for the Twitter settings. To help you find it, you can search for the term twitterUsername. Change the Twitter username to yours.

twitterUsername = 'your_username',

You can also change the number of Tweets to suit. Change the code:

tweetCount = 2,

to whatever number you want. The Twitter feed popup box should now show your own tweets.

You can change the colour scheme by searching for <!--Start Outside--> in index.html, in class of the div below the comment you can add your colour class from the following list. lime is the default so replace it with another.

Align left / right / center

You can choose to align your template either in the center (default) to the left or to the right. You can change the alignment by simply adding the class center, left or right into the body tag on the index.html page.

<body class="dark right">

Rounded corners

You can choose to have CSS3 rounded corners on your template. You can change to rounded corners by simply adding the class rounded into the body tag on the index.html page.

<body class="dark right rounded">

Show / hide content footer

You choose to show or hide the footer beneath the content area with mini social network icons within it. To show it add the class show-footer into the body tag on the index.html page. If you do not want it there do not add anything - it is hidden by default.

<body class="dark right rounded show-footer">

Adding background images

The backgrounds on the preview version of Launch Pad are not included in the downloadable version due to copyright reasons, so you will need to add your own background images. You are free to download the images that we used and use them yourself, see the section below called Our live preview background images. We suggest making the background images around 1680px width x 1050px height. This will ensure that the image looks great in almost all screen resolutions. To keep the download size relatively small we found that choosing a jpeg quality between 40 - 60% meant that the file size was about 250kb - 300kb each.

Add your background images to the folder images/backgrounds/. After that you will need to add links to your images in js/scripts.js, at the top of that file you will find the following code:

var backgrounds = [
    'images/backgrounds/placeholder.jpg'
],

Add the paths to your images one on each line with a comma at the end of every line except the last one, the file name itself is not important as long as it points to the background, e.g.:

var backgrounds = [
   'images/backgrounds/your_background1.jpg',
   'images/backgrounds/your_background2.jpg',
   'images/backgrounds/your_background3.jpg',
   'images/backgrounds/your_background4.jpg',
   'images/backgrounds/your_background5.jpg',
   'images/backgrounds/your_background6.jpg'
],

We have included a CSS based full screen background fallback for users with JavaScript disabled. You will need to change index.html, search for the term <!-- Non-JavaScript background --> you'll see an <img> tag that is pointing to the placeholder background. Change this to point to your background image e.g.

<img src="images/backgrounds/your_background1.jpg" alt="" />

Background script options

There are some options you have to change the functionality of the background image script:

KeyDefaultDescription
speedIn3000Speed of the "fade in" transition between background images in milliseconds.
speedOut3000Speed of the "fade out" transition between background images in milliseconds.
synctrueIf true - both fade animations happen at the same time, if false - "fade in" waits for "fade out" to complete.
minimiseSpeedIn1000Speed that the website fades in when the close button is clicked.
minimiseSpeedOut1000Speed that the website fades out when the maximise button is clicked.
controlSpeedIn500Speed that the controls fade in at in fullscreen mode.
fadeIEfalseFade the website when the minimise button is clicked in IE 7,8.
preloadtruePreload the background images for faster loading.
savetrueSave the current background image across pages in a cookie.
slideshowtrueEnable the slideshow functionality (play/pause button).
slideshowAutotrueStart the slideshow automatically.
slideshowSpeed7000The time that each background image is displayed in the slideshow.
randomfalseDisplay the background images in random order each time, forces "save: false".
keyboardtrueUse the keyboard to control the background images. Left arrow = prev, right arrow = next, esc = close fullscreen mode.
centerXtrueIf true, the background will be horizontally centered in the window. If false it will be left aligned.
centerYtrueIf true, the background will be vertically centered in the window. If false it will be top aligned.
onLoadfalseCallback that fires right before loading a background image.
onCompletefalseCallback that fires when the background image has finished loading (and animating).

To use the options, open scripts.js and find the code backgroundOptions = {, it's near the top of the file. You should see code looking like the following:

backgroundOptions = {

},

Add your own options between the curly brackets as shown below, each option you add should end in a comma, except the last one, for example

backgroundOptions = {
    speedOut: 0,
    slideshow: false
},

Note: make sure to end each line of your options with a comma (except the last line), or the script will break!

Note: the website fade in / fade out effect has been changed to a show / hide effect in Internet Explorer since the way the browser handles the fading of some positioned elements is not pretty.

Different default background per page

It's possible to set a different default background on a per page basis. You will need to find out the index of the image in your list of background images that you added to scripts.js. The first background in your list has the index 0, the second one has the index 1, the third image has index 2, and so on. Once you have the index, add the code below to your page (use your number instead of 3):

<script type="text/javascript">var backgroundIndex = 3;</script>

Add your code before this line in the <head> of the page:

<script type="text/javascript" src="js/scripts.js"></script>

Different backgrounds per page

It's possible have an entirely different set of background images on a per page basis. The code you need to create the backgrounds is the same as inside scripts.js but you need to add it in the <head> of page instead.

Gallery customisationTop

Changing image height

Note: We highly recommend not altering the width of the gallery. However you may wish to change the height in which case you will need to locate and change the heights of two elements in the styles.css the Gallery section.

You will need to change the hover image height and the image wrapper height. Locate .threecol-portfolio .one-portfolio-item and .threecol-portfolio .portfolio-hover.

Take note of your new image height and do the following sum to calculate the new heights of these CSS elements:

.one-portfolio-item - Image height (px) + 6px = height | Eg: Img height = 92px (+ 6px) = 98px

.portfolio-hover - Image height = same as image height.

Fancybox

You will need to create 2 sizes of image for each of the images you want to display in your portfolio, one for the thumbnail and one for the popup size. We recommend that you make the popup size have a maximum height of 500px. As a guide, our placeholder images are 800px wide. The size of the thumbnail is:

  • 148px width by 92px height

You should save your thumbnail images into the folder images/gallery/ and pop up images to images/gallery/popup

Now open the index.html file and search for the term <!--Gallery content-->, this should take you to just before the list of images. Each portfolio item looks similar to this:

<li class="one-portfolio-item">
    <a class="portfolio-thumb-link" href="images/gallery/popup/placeholder.jpg" rel="portfolio">
        <img src="images/gallery/placeholder.png" alt="" />
    </a>
</li>

As you can see, it consists of an image surrounded by a link, the image is the thumbnail, and the link points to the popup image. You now need to change the src="" attribute of the <img> to point to your thumbnail image. Then change the href="" of the <a> to point to the popup image. For example, your code might look like this once you've made the change, if your thumbnail and popup image were both called photo1.jpg:

<li class="one-portfolio-item">
    <a class="portfolio-thumb-link" href="images/gallery/popup/photo1.jpg" rel="portfolio">
        <img src="images/gallery/photo1.jpg" alt="" />
    </a>
</li>

Repeat these steps for each portfolio item you want to add. You may need to delete some placeholder images, so just delete the entire <li></li> surrounding it.

TroubleshootingTop

Background shows a blank screen

Most likely the background image does not exist, check the path to the image in scripts.js and check that it exists on the server.

Poor background animation performance

Old browsers may not be able to handle the animation between background images at the default settings. You can change the JavaScript options (in js/scripts.js near the top) to be more forgiving as shown below.

backgroundOptions = {
    speedOut: 0,
    sync: false
},

The options make it so that when you click next or prev, it hides the current background instantly and then starts fading in the next background, instead of fading in and out both images at the same.
